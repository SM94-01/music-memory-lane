# Android / Firebase Cloud Messaging setup

The `android/` native project is generated by Capacitor — do not commit it
from Lovable. Run these commands **locally** after cloning:

```bash
bun install
bun run build
npx cap add android          # first time only
npx cap sync android
```

## 1. Firebase console

1. Create (or open) a Firebase project.
2. Add an Android app with package id **`app.lovable.trax`**
   (must match `appId` in `capacitor.config.ts`).
3. Download **`google-services.json`**.

## 2. Drop `google-services.json` in place

Copy the file to:

```
android/app/google-services.json
```

`@capacitor/push-notifications` (v8) already wires the Google Services
Gradle plugin and the `firebase-messaging` dependency into the generated
Android project, so no manual Gradle edits are required beyond adding the
file.

## 3. Android 13+ runtime permission

The `POST_NOTIFICATIONS` permission is added automatically by the plugin.
The app requests it at runtime on first launch via
`initPushNotifications()` (see `src/lib/push/index.ts`).

## 4. Verify

```bash
npx cap run android
```

On launch you should see a `[push] ...` log with the FCM token, and a new
row in the `device_tokens` table for the signed-in user.
